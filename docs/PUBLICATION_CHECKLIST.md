# Publication Checklist

Before making a repository public:

- [ ] No real transcripts.
- [ ] No real board data.
- [ ] No real email addresses except examples.
- [ ] No real Google document URLs.
- [ ] No real database URLs.
- [ ] No real OAuth client IDs or client secrets.
- [ ] No real model-provider API keys.
- [ ] No real bearer tokens.
- [ ] No company-specific project names.
- [ ] `.env.example` contains placeholders only.
- [ ] Privacy, Limited Use, and Security docs are present.
- [ ] The README says humans must be authorized to process source data.
- [ ] The model-provider path is pluggable.
- [ ] Local Claude Code mode, if included, is marked personal/internal.
- [ ] Automated candidates land in `Automated`.
- [ ] Backups and restore behavior are documented.
- [ ] The setup prompt tells agents not to print secrets.
